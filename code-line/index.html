Grid Layout for Projects


<section class="projects-grid">
  <article class="project-item">
    <h3>Project Title 1</h3>
    <p>Short description of the project.</p>
  </article>

  <article class="project-item">
    <h3>Project Title 2</h3>
    <p>Short description of the project.</p>
  </article>

  <article class="project-item">
    <h3>Project Title 3</h3>
    <p>Short description of the project.</p>
  </article>
</section>

The <section> element groups related content, here the projects.

The class projects-grid is used to apply CSS grid or flexbox for layout.

Each project is an <article> with a title and a short description.

This semantic grouping helps both accessibility and SEO.

CSS will handle the grid layout to create responsive columns or rows.

Image Grid Section


      <div class="grid">
        <a href="#wonderwall" class="grid-item">
          <img src="img/wonderwall.webp" alt="Wonderwall project preview" loading="lazy">
        </a>
        <a href="#video" class="grid-item">
          <img src="img/video-thumbnail.webp" alt="Video project preview" loading="lazy">
        </a>
        
      </div>

The <div class="grid"> acts as a container for a responsive grid layout of linked images.

Each <a> element represents a clickable grid item linking to a section or project anchor.

Inside each <a> is an <img> tag with a thumbnail image. The loading="lazy" attribute defers image loading until near viewport, improving page speed.

This approach creates a clean, navigable image gallery with semantic HTML and easy styling via CSS grid or flexbox.

Video Container Section


<div class="video-container">
  <video autoplay muted loop playsinline preload="metadata">
    <source src="video/project-preview.webm" type="video/webm">
    <source src="video/project-preview.mp4" type="video/mp4">
    Your browser does not support the video tag.
  </video>
</div>

The <div class="video-container"> acts as a wrapper to style and position the video element.

The <video> tag embeds the video directly into the page. It uses these attributes:

  • autoplay: starts playing automatically.
  • muted: silent playback (required by many browsers for autoplay).
  • loop: repeats the video indefinitely.
  • playsinline: allows inline playback on mobile devices.
  • preload="metadata": loads only metadata initially to improve performance.

Multiple <source> elements provide fallback video formats for better browser support.

The fallback text inside <video> tags informs users with unsupported browsers.

The <div class="header"> groups the site's top area containing your name and contact information.

This structure is simple and semantic, making it easy to style and accessible.

Image with Caption / Responsive Width


        <figure class="image-container" style="width: 50vw;">
          <img src="img/path-to-image.webp" 
            alt="Short but descriptive alt text" 
            class="responsive-image" 
            loading="lazy">
          <figcaption class="caption">
            Project title, Location<br>
            Description
          </figcaption>
        </figure>

<figure> creates a semantic container that groups an image and its caption. The class="image-container" allows styling with CSS, and the inline style width: 50vw; sets its width to 50% of the viewport width, making it responsive.

The <img> tag displays the image. The src attribute should point to your actual image path. The alt attribute provides a short but descriptive alternative text important for accessibility and SEO. The class="responsive-image" is for styling, and loading="lazy" defers loading the image until it's near the viewport to improve performance.

The <figcaption> tag contains the caption linked semantically to the image. The class="caption" is for styling the caption text. The <br> creates a line break between the project title and the description.

The </figure> closes the container.
